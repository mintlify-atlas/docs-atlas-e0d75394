Decompose is a Kotlin Multiplatform library that breaks your application code into tree-structured, lifecycle-aware business logic components — commonly called BLoCs. Each component encapsulates its own logic and lifecycle, while UI is entirely optional and plugged in from outside, enabling true separation of concerns across Android, iOS, Desktop, and Web platforms.

## Why Decompose?

Decompose takes a different approach to multiplatform architecture. Instead of making UI the orchestrator of navigation, Decompose puts business logic in charge. Components form a strict parent-child tree — each parent only knows its immediate children — which enables:

- **Pluggable UI** — connect Compose, SwiftUI, React, or Android Views to the same component
- **Shared navigation logic** — navigate in pure Kotlin, no platform-specific code required
- **Testable business logic** — unit-test components without instrumentation or UI frameworks
- **Lifecycle awareness** — every component has its own lifecycle, automatically managed

## Key concepts

<AccordionGroup>
  <Accordion title="Component">
    A component is just a normal Kotlin class — no base class required. It encapsulates business logic and optionally exposes state. UI is plugged in externally, making components fully testable without any UI framework.
  </Accordion>
  <Accordion title="ComponentContext">
    Every component receives a `ComponentContext` that provides lifecycle, state preservation (`StateKeeper`), instance retaining (`InstanceKeeper`), and back-button handling (`BackHandler`). Use delegation to bring all these APIs into your component's scope.
  </Accordion>
  <Accordion title="Configuration">
    Each child component is identified by a persistent `@Serializable` configuration class that holds its arguments. Decompose automatically saves and restores configurations across process death and configuration changes.
  </Accordion>
  <Accordion title="Value / MutableValue">
    Decompose provides its own observable state holder `Value<T>` that works with Compose, SwiftUI, and Kotlin/React without requiring coroutines. Convert to `StateFlow` or `Flow` as needed.
  </Accordion>
</AccordionGroup>
