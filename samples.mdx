Decompose ships with a comprehensive set of sample applications that demonstrate real-world usage across every supported platform and UI framework. All samples are available in the [main repository](https://github.com/arkivanov/Decompose).

## Multi-Feature Sample App

The main sample is a full-featured multiplatform application that demonstrates the breadth of Decompose's capabilities across Android, iOS, Desktop, and Web.

### Features demonstrated

- Nested reusable components
- Bottom navigation
- Nested navigation
- All navigation models: Child Stack, Child Slot, Child Pages, Child Items, Generic Navigation
- State preservation using `StateKeeper`
- Retaining instances using `InstanceKeeper`
- Pluggable UI: Android Views, Compose, SwiftUI, Kotlin/React
- [Play Feature Delivery](https://developer.android.com/guide/playcore/feature-delivery) for Android
- Tests including Compose UI tests

### Component hierarchy

The shared module contains the following component tree:

- **Root** — application root
  - **Tabs** — bottom navigation bar with selected tab
    - **Menu** — buttons leading to fullscreen components
    - **Counters** — stack of `CounterComponent` instances, each counting independently
    - **Cards** — draggable card stack controlled by gestures
    - **Multi-Pane** — list/details in single-pane or side-by-side layout
  - **Dynamic Features** — Play Feature Delivery on Android, direct dep on other platforms
  - **Custom Navigation** — Generic Navigation model example
  - **Pages** — Child Pages navigation example
  - **Shared Transitions** — Compose shared element transitions with Decompose

### Integration tests

The sample includes multiplatform integration tests that run without instrumentation:

- `RootComponentIntegrationTest` — navigation tests for the root component
- `TabsComponentIntegrationTest` — navigation tests for the tabs component
- `CountersComponentIntegrationTest` — navigation tests for the counters stack
- `CounterComponentTest` — unit tests covering instance retaining and state preservation
- `CounterContentTest` — Compose UI tests for the counter screen

## Sample Todo App

A simple todo list application with list and detail screens, sharing UI between Android and Desktop via Compose Multiplatform, with SwiftUI on iOS.

Libraries used:
- [MVIKotlin](https://github.com/arkivanov/MVIKotlin) — presentation and business logic
- [Reaktive](https://github.com/badoo/Reaktive) — background processing and data transformation
- [SQLDelight](https://github.com/cashapp/sqldelight) — data storage

[View on GitHub](https://github.com/IlyaGulya/TodoAppDecomposeMviKotlin)
