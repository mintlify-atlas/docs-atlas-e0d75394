title: "Child Pages: Pager and Tab Navigation in Decompose KMP"
description: "Implement pager and tab navigation in Decompose with PagesNavigation and childPages(). Covers select, selectNext, selectPrev, setItems, and page lifecycle."
---

Child Pages is the Decompose navigation model for displaying an ordered list of child components with exactly one selected (active) component at a time. The list can be empty. It is the natural fit for horizontal pagers, tab bars, and any navigation where users swipe or tap between several peer screens.

Unlike Child Stack, there is no concept of "going back"—selection is direct by index or relative (next/previous). The selected page is `RESUMED`, its immediate neighbours are `STARTED` (so they can preload), and all other pages are `DESTROYED`.

## Core concepts

Child Pages introduces three types:

- **`Pages<C>`** — the raw navigation state: `items` (the list of all configurations) and `selectedIndex`.
- **`ChildPages<C, T>`** — the observable output: `items` (a list of `Child` wrappers pairing each configuration with its component instance or `null`) and `selectedIndex`.
- **`PagesNavigation<C>`** — the mutable handle you call navigation operations on. It implements `PagesNavigator<C>`, which exposes a low-level `navigate(transformer, onComplete)` method.

### Lifecycle of pages

| Position relative to selection | Lifecycle state |
|---|---|
| Selected page | `RESUMED` |
| Direct neighbours (index ± 1) | `STARTED` |
| All other pages | `DESTROYED` |

<Note>
  You can override the default lifecycle assignment by implementing a custom `pageStatus` parameter in `childPages`. The source code of the built-in implementation is a good starting point.
</Note>

## Setting up Child Pages

<Steps>
  <Step title="Create a PagesNavigation instance">
    ```kotlin
    private val navigation = PagesNavigation<Config>()
    ```
  </Step>
  <Step title="Call childPages() on ComponentContext">
    ```kotlin
    val pages: Value<ChildPages<*, PageComponent>> =
        childPages(
            source = navigation,
            serializer = Config.serializer(), // null to disable state saving
            initialPages = {
                Pages(
                    items = List(5) { index -> Config(data = "Page $index") },
                    selectedIndex = 0,
                )
            },
        ) { config, childComponentContext ->
            DefaultPageComponent(
                componentContext = childComponentContext,
                data = config.data,
            )
        }
    ```
  </Step>
  <Step title="Observe in the UI">
    Subscribe to `Value<ChildPages>` and pass the result to a `HorizontalPager` or `TabRow`. Use the `ChildPages` composable from the Decompose Compose extensions for the pager integration.
  </Step>
</Steps>

### childPages() parameters

<ParamField path="source" type="NavigationSource<PagesNavEvent<C>>" required>
  The `PagesNavigation` instance that drives this model.
</ParamField>

<ParamField path="serializer" type="KSerializer<C>?">
  A kotlinx-serialization serializer for the configuration type. Pass `null` to opt out of navigation state persistence.
</ParamField>

<ParamField path="initialPages" type="() -> Pages<C>" required>
  Factory that returns the initial `Pages` state, including the full list of configurations and the initially selected index.
</ParamField>

<ParamField path="key" type="String" default="DefaultChildPages">
  Unique key within the parent component. Required when you have two or more Child Pages models in the same parent.
</ParamField>

<ParamField path="childFactory" type="(C, ComponentContext) -> T">
  Factory function called to create each child component.
</ParamField>

## Navigation operations

### select(index)

Selects the component at the given index. Throws `IllegalArgumentException` if the index is out of bounds.

```kotlin
// [A*, B, C] --select(2)--> [A, B, C*]
navigation.select(index = 2)
```

### selectNext(circular)

Selects the next component. When `circular = true` and the last page is selected, wraps around to the first page. Defaults to `circular = false`.

```kotlin
navigation.selectNext()              // stops at the end
navigation.selectNext(circular = true) // wraps around
```

### selectPrev(circular)

Selects the previous component. When `circular = true` and the first page is selected, wraps around to the last page.

```kotlin
navigation.selectPrev()
navigation.selectPrev(circular = true)
```

### selectFirst / selectLast

Convenience functions that jump to the first or last component in the list.

```kotlin
navigation.selectFirst()
navigation.selectLast()
```

### setItems(transformer)

Replaces the item list using a transformer function. The `selectedIndex` is automatically clamped to remain within the new list's bounds.

```kotlin
// Append a new page
navigation.setItems { items -> items + Config(data = "New page") }

// Remove a page
navigation.setItems { items -> items - Config(data = "Page 3") }
```

<Note>
  `setItems` is available since version `3.4.0-alpha02`.
</Note>

### clear

Removes all components from the list, leaving `items` empty.

```kotlin
navigation.clear()
```

## Full example

<CodeGroup>

```kotlin PageComponent.kt
import com.arkivanov.decompose.ComponentContext

interface PageComponent {
    val data: String
}

class DefaultPageComponent(
    componentContext: ComponentContext,
    override val data: String,
) : PageComponent, ComponentContext by componentContext
```

```kotlin DefaultPagesComponent.kt
import com.arkivanov.decompose.ComponentContext
import com.arkivanov.decompose.router.pages.ChildPages
import com.arkivanov.decompose.router.pages.Pages
import com.arkivanov.decompose.router.pages.PagesNavigation
import com.arkivanov.decompose.router.pages.childPages
import com.arkivanov.decompose.router.pages.select
import com.arkivanov.decompose.value.Value
import kotlinx.serialization.Serializable

interface PagesComponent {
    val pages: Value<ChildPages<*, PageComponent>>

    fun selectPage(index: Int)
}

class DefaultPagesComponent(
    componentContext: ComponentContext,
) : PagesComponent, ComponentContext by componentContext {

    private val navigation = PagesNavigation<Config>()

    override val pages: Value<ChildPages<*, PageComponent>> =
        childPages(
            source = navigation,
            serializer = Config.serializer(),
            initialPages = {
                Pages(
                    items = List(10) { index -> Config(data = "Item $index") },
                    selectedIndex = 0,
                )
            },
        ) { config, childComponentContext ->
            DefaultPageComponent(
                componentContext = childComponentContext,
                data = config.data,
            )
        }

    override fun selectPage(index: Int) {
        navigation.select(index = index)
    }

    @Serializable
    private data class Config(val data: String)
}
```

</CodeGroup>

## Duplicate configurations

By default, every configuration in the `items` list must be unique by equality. To allow duplicate configurations, enable the global opt-in:

```kotlin
import com.arkivanov.decompose.DecomposeSettings

DecomposeSettings.duplicateConfigurationsEnabled = true
```

<Note>
  Duplicate configuration support is stable since version `3.5.0-alpha01`.
</Note>

## Multiple Child Pages in one component

Each Child Pages model in the same parent component must have a unique `key`:

```kotlin
private val horizontalPages = childPages<HPageConfig, HPageChild>(
    source = hNavigation,
    key = "HorizontalPages",
    // ...
)

private val verticalPages = childPages<VPageConfig, VPageChild>(
    source = vNavigation,
    key = "VerticalPages",
    // ...
)
```
